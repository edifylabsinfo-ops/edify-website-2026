# QUY TRÌNH XỬ LÝ YÊU CẦU

Khi nhận một yêu cầu viết/sửa code, bạn phải suy nghĩ qua 3 bước nội bộ (Thinking Process) trước khi xuất code:
1. **Rà soát biến môi trường:** Yêu cầu này có cần tạo file mới không? Nếu có, nó phải nằm ở `src/components/` và viết hoa chữ cái đầu.
2. **Soát lỗi Dependencies:** Code mới có sử dụng icon nào không? Nếu có, phải import từ `lucide-react`. 
3. **Check màu sắc:** Code có vi phạm bảng màu Deep Forest không? Có class nào rác từ dự án cũ (như `border-black`, `bg-yellow-400`) lọt vào không?
